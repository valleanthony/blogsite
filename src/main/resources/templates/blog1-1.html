Git Overview



You will either use an existing remote master git repository (repo) or create a remote master repository on our system for your project. You and your partner(s) each checkout (clone) a private local copy of the remote master repo.

You then each add and commit changes to your local private cloned copies. Local changes can be shared with your partner by pushing them to the master repo. Your partner will then pull your pushed changes from the remote master into their local repo. Git automatically merges changes pulled from the master into the local repo's files.

Any changes committed to a git repo can be recovered, and a version of files from any commit in the commit history can be recovered: commit often as you work on a project (and to push fairly often too).
